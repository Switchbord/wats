## WATS-97 webhook media ID retention

Current behavior: webhook media IDs received via webhook are downloadable for 7 days after 2025-10-09. Linear WATS-97 source evidence is the WhatsApp changelog dated 2025-09-24; Meta reduced the current webhook media ID downloadability window from 30 days to 7 days.

Parity interpretation: WATS documents and exposes the media IDs/URLs needed for retrieval, but applications that need media after the retention window should promptly download it and persist it in durable storage. This is documentation/guard coverage only; it does not add automatic media persistence or change upload/download byte limits.

## Post-foundations roadmap

The following work is explicitly **out of scope** for the foundations pivot and requires new post-pivot work (much of it also requires live credentials that the pivot does not touch). This section is the shortlist of what WATS consumers **cannot** use today.

### Message composer follow-ups after WATS-38
- live Meta credential checks for WATS-38 composer variants
- generic/raw unsupported passthrough ergonomics if needed by consumers
- order-specific outbound helpers if Meta exposes a supported send path in the target API version

### Templates
WATS-39 implements credential-free SDK coverage for list/create/get/update/delete template requests, approved template send payloads, component builders, parameter-count validation, and template status/quality/category/components webhook filtering. Remaining template work is credential-gated live Meta validation and any future high-level library-template/bulk-authentication helpers.

### Flows
WATS-40 implements credential-free SDK coverage for Flow list/get/create/update metadata/update JSON/publish/delete/deprecate/assets requests, WABAClient methods, bounded Flow JSON validation, and data-exchange response builders. Remaining Flow work is live Meta validation, production Flow hosting/deployment, encrypted request decrypt/encrypt handling, metrics, migration, and deeper webhook/filter ergonomics where needed.

### Calling API
WATS-41 implements credential-free SDK coverage for initiate/pre-accept/accept/reject/terminate request shapes, typed calling webhook variants, and calling typed filters. Remaining calling work is live Meta validation with real phone numbers/call sessions and any deeper WebRTC/media signaling or production call orchestration beyond the stable Graph request/response/webhook surfaces.

### Admin (business management)
WATS-42A implements read-only business/admin inventory: WABA info, subscribed-app listing, enhanced WABA phone-number listing, phone-number info/settings, business profile reads, and commerce settings reads. Credential-free MockTransport only. Remaining admin work is credential-gated or separate-slice: WABA CRUD, phone-number registration/configuration mutations, subscribed-app mutations/callback overrides, token flows, display-name/profile/settings/commerce updates, catalog/product management, and analytics.

### Media credential-gated validation (WATS-37)
- live Meta checks for upload / metadata / binary download / delete / decrypt / upload-session behavior

### Rate-limit retry / backoff (WATS-86)
- `@wats/graph` ships opt-in `createReliableTransport(inner, policy)` for ky-like retries/backoff/timeouts without an external dependency.
- Retries transient `GET`/`DELETE` failures (`GraphNetworkError`, HTTP `429`, HTTP `5xx`) with bounded exponential full-jitter backoff, honors `Retry-After`, and composes per-attempt `timeoutMs` with caller `AbortSignal`.
- Non-idempotent `POST`/`PUT`/`PATCH` failure retries stay off by default to avoid duplicate WhatsApp sends; HTTP `429` is the narrow rate-limit exception across methods. Per-endpoint concurrency caps remain future work.

### Observability standardization
- OpenTelemetry span emission from `TypedRouter` + `GraphClient`
- metric names stabilized (dispatch.latency / requests.inflight / errors.code)
- context propagation across handlers + listeners

Consumers who need any of the above should open a new post-pivot Linear issue (the WATS-1..29 backlog is closed with this release) and gate behind a `credentialed` environment.
